使用OPEN GPG证书加密任何文件

· 2020-09-05 · # 安全 # 证书

前文我们介绍了S/MIME加密，本文我们来尝试另一种更自由，全面的OPEN GPG加密。

准备

需要两个软件Kleopatra(gpg4win)和Mozilla Thunderbird

Kleopatra

  1. 前往官网https://www.gpg4win.org/
    或蓝奏云下载
  2. 一路默认安装完成

Mozilla Thunderbird

  1. 前往官网https://www.thunderbird.net/zh-CN/下载
  2. 一路默认安装完成

创建密钥对

  1. 打开Kleopatra
  3. 输入个人信息，如有需要可以调高密钥安全等级

我并不建议创建无时间限制的证书，这将导致一些不必要的麻烦，建议使用默认的2年期限

  1. 输入用于加密私钥的密码
  2. 完成
  3. 导出私钥

使用Mozilla Thunderbird与你的朋友发送加密邮件

  2. 选中密钥
  3. 给你的朋友发送一封邮件，选择【对此消息数字签名】与【附上我的公钥】

对方视角，感谢ZiKin提供的协助
导入公钥
选择信任证书

  1. 导入对方证书
  2. 信任对方证书
  3. 信息安全正常

Mozilla Thunderbird的证书管理


在此能够选择信任对方的证书

Kleopatra导入其他人的公钥

公钥能够使别人加密成只有你能能打开的加密文件，打个比方，公钥是锁，私钥是钥匙，只有私钥能够解密你的消息。

  1. 让你的朋友导出公钥
  2. 你导入他的公钥

使用Kleopatra加密其他文件

  2. 如果不选择加密，文件将能被任何人打开，并得知：你是文档的作者、文档未被其他人修改过

效果

  1. 如果选择加密，文件仅能由你打开(拥有证书+私钥的人)

效果

  1. 如果选择密码加密，文件能够由拥有你所选择公钥的拥有者和拥有密码的人打开

效果

思考

  1. 相比于依赖证书颁发机构(CA)的SSL/TLS / S/MIME证书，OPEN GPG能够自主控制证书状态，有效期，吊销
  2. 由于①，你需要验证你获取的【公钥】是否真的是你朋友发给你的，而不是被篡改的

DNS ＆ CDN 科普

· 2020-04-15 · # 科普

域名与通常所熟知的IP地址是由DNS进行连接的，本章，让我们来学习

简介

  • IP 互联网中唯一的通讯地址，目前有IPV4和IPV6两个版本，v4已经于19年分完
  • 域名 通过DNS解析到IP，使地址容易记忆
  • DNS 将域名转换为IP地址的服务器
  • CDN 内容分发技术，令用户的访问更快

说明

域名解析类似于把你的域名与其他信息绑定发送给解析服务器，每次浏览器访问域名都会向解析服务器发送请求，访问所绑定的资源。

记录类型一览表

仅列出常用功能

解析类型 描述 功能
A IPv4地址已记录 绑定IPv4地址
AAAA Pv6地址已记录 绑定IPv6地址
CNAME 别名记录 将绑定的域名解析到另一个域名
NS 名称服务器记录 委托DNS区域（DNS zone）使用已提供的权威域名服务器
MX 电子邮件交互记录 引导域名到该域名的邮件传输代理（MTA, Message Transfer Agents）列表

子域名表达式

表达符 意义 其他
@ 指向根域名 常用于绑定主页与邮箱
* 指向所有未被解析过的域名 用户输入错误的子域名，将其引导到主页
www 约定俗成 常用于绑定主页
任何合法字符串 自定义 自定义

域名的构成

  • 本站域名为：parksi.xyz叫做顶级域名，xyz即是后缀,解析到www子域
  • 本站的实际域名为：www.parksi.xyz叫做二级域名，后缀是xyz,二级域名是www
  • 本站邮箱域名为 ms.parksi.xyz,邮箱构成为：用户名@邮箱域名，我的邮箱就是parksi@ms.parksi.xyz, 由MX记录ms子域到邮箱服务器实现
    总结：
    最后一个点后的即是后缀
    最后一个点前每一个点都是一级域名
